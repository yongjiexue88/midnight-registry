# 午夜登记簿 (Midnight Registry) - 玩法拓展评估报告

本报告基于“身份核验恐怖游戏”的核心理念，对潜在的玩法拓展进行了评估和整理。核心原则：**新玩法必须服务于“核验身份、制造压力、推进剧情”**，避免偏离门岗体验的核心。

## 推荐加入的核心玩法拓展

这些玩法高度契合“门岗调查”体验，建议按优先级逐步加入：

### 1. 办公室设备故障 (制造压力与互动)
- **概念**：夜班期间，关键设备（电话、CCTV、ID扫描器、门锁、灯光）随机故障。
- **机制**：玩家需要在处理访客的同时，通过简单小游戏（如连线、调频、输密码）修复设备。
- **价值**：不偏离核心，且能有效增加紧张感。

### 2. CCTV 监控深度交互 (调查升级)
- **概念**：将现有的CCTV系统升级为多镜头可切换、可回放的交互系统。
- **机制**：切换频道（大厅、走廊、电梯等），暂停、放大、截图寻找诡异线索（如无影子、同时出现、不存在的楼层）。
- **价值**：极大地丰富了调查手段，适合中期版本。

### 3. 提问卡牌系统 (策略性问询)
- **概念**：将固定的提问按钮改为带有次数限制的“问题卡牌”（身份、关系、记忆、陷阱卡）。
- **机制**：每位访客限用3张卡，玩家需在每日班前配置卡组。
- **价值**：增加提问的策略性和组合深度，开发成本可控。

### 4. 档案污染与修复 (剧情与核心循环结合)
- **概念**：随着游戏进行（如第5夜），真实档案会被伪人污染篡改。
- **机制**：玩家需要收集旧照片、录音、备注等原始资料，对比并恢复真实档案。
- **价值**：将“查档案”升华为“保护真实历史”，契合诡异主题。

### 5. 案件板 / 证据拼图 (推理可视化)
- **概念**：将当前的证据标记升级为可视化的案件板。
- **机制**：玩家将发现的矛盾点（ID错误、行为异常、监控异常）拖入案件板。证据链完整度决定了玩家可以采取的应对措施（如呼叫安保）。
- **价值**：让玩家的决策基于实质推理，而非盲目猜测。

---

## 其他有潜力的玩法 (视开发进度考虑)

- **访客排队压力**：引入等待时间与住户耐心，增加决策时间压力（建议逐步引入）。
- **白天准备阶段**：班前阅读报告、升级设备、选择卡组，增加局外成长感。
- **住户关系网**：从单点核查扩展到关系核查（谁认识谁），后期关系网可被伪人污染。
- **声音识别**：对比访客语音与档案录音的特征（语速、情绪、背景音）。
- **特殊访客事件**：引入双胞胎、紧急伤员、失忆住户等打破常规流程的特殊考验。
- **办公室设备升级**：通过游戏进度解锁更高级的扫描仪、回放功能等（需平衡代价）。
- **资源管理**：限制电话、扫描或呼叫安保的次数。
- **轻量级夜间巡逻**：点击式场景切换检查电箱或后门，伴随离岗风险（仅限极简交互，避免变动作游戏）。

---

## 坚决不建议加入的玩法

以下玩法会严重稀释“门岗调查”的核心体验，应避免加入：
- ❌ 自由移动大地图 / 开放世界探索
- ❌ 动作战斗 / 追逐逃跑
- ❌ 复杂的物品合成
- ❌ 多人联机
- ❌ 农场种田 / 抽卡养成 / 大量 RPG 装备

---

## 版本规划建议 (Roadmap)

推荐采用迭代方式逐步丰富核心循环，避免一次性堆砌所有系统：

*   **V1 (基础核验)**: 访客出现 ➔ 查档案 ➔ 放行/拒绝 ➔ 基础结算。
*   **V2 (调查强化)**: 引入电话确认、提问系统、文字CCTV、证据标记、每日规则、出入记录。
*   **V3 (深度与变数 - 当前重点)**: 升级交互式CCTV监控、加入提问卡牌、引入设备故障机制、开启档案污染与修复、特殊访客事件。
*   **V4 (外围系统与重玩性)**: 白天准备阶段、建立住户关系网、办公室升级、引入无限夜班 (Endless Shift) 和随机挑战模式。

## 结论

未来所有的玩法拓展，都必须能够无缝接入当前的核心工作流，不应游离于系统之外：
`访客来访 ➔ 玩家检查文件 ➔ 发现疑点 ➔ 运用工具(可能故障/矛盾) ➔ 标记证据 ➔ 做出决断 ➔ 结果影响`

**当前最优先开发项：**
> **CCTV 可交互化 + 提问卡牌 + 设备故障 + 档案修复。**
