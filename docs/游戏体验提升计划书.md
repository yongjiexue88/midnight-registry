游戏音效不要只当“氛围”，它应该成为**玩法提示 + 情绪压力 + 裁决反馈**。你这个游戏最适合做成一种声音体验：

> 雨声、旧电话、电流、纸张、印章、监控噪点、门外敲玻璃声，组成一个“夜班前台核验仪式”。

你目前已经有音频方向：雨声循环、值班铃、电话铃、拨号、接通、死寂、敲门、扫描通过/错误、CCTV 冻结、印章、安保警报，以及安全/声誉/稳定度受损音效。这个基础是对的，但还需要进一步系统化。

---

# 一、音效设计核心原则

你的音效要服务三个目标：

## 1. 让玩家知道自己做了什么

每个按钮都要有明确声音反馈。

比如：

```text
打开证件：纸张抽出声
打开档案：厚纸翻页声
点击电话：旧电话拿起声
扫描证件：低频电流 + 哔声
保存证据：图钉/夹子“咔哒”
盖章：沉重印章声
```

玩家每点一下，都应该感觉自己真的在操作前台设备。

## 2. 让玩家知道事情是否正常

同一个工具要有“正常版”和“异常版”。

比如扫描器：

```text
扫描通过：短促、干净、明亮的 beep
扫描异常：beep 断裂 + 电流噪音
扫描伪通过：表面正常，但尾音有轻微倒放/失真
扫描器损坏：持续电流 + 卡壳声
```

这会让玩家慢慢学会听声音判断危险。

## 3. 让玩家感到裁决有重量

盖章不能只是 UI 按钮。
它应该像在决定一个人的命运。

```text
允许进入：印章声 + 门锁打开 + 远处脚步进入楼道
拒绝进入：印章声 + 门外沉默 + 雨声变大
呼叫安保：印章声 + 红灯警报 + 对讲机噪音
留置等待：印章声变轻 + 时钟声变明显 + 来客呼吸声
错误裁决：印章声之后出现不该出现的低频噪音
```

# 二、整体声音风格

你的游戏不是现代科幻，也不是 jumpscare 恐怖。
更适合这个关键词：

```text
老旧
潮湿
纸质
电流
低频
压抑
仪式感
半真实半诡异
```

可以参考这种声音质感：

```text
旧公寓雨夜
90 年代前台电话
老式监控录像
纸张档案室
坏掉的荧光灯
远处电梯运行
湿鞋踩地砖
玻璃外的敲门
```

不要做太多“怪物吼叫”。
这个游戏恐怖点应该是：**一切听起来都像正常办公，但又慢慢不对劲。**

# 三、主要音效分类

## A. 环境底噪

这是游戏的底层情绪。

### 1. 雨声循环

雨声是主背景。
建议做 3 个版本：

```text
rain_light_loop.wav      第一夜、第二夜
rain_medium_loop.wav     第三夜、第四夜
rain_heavy_loop.wav      第五夜以后
```

随着夜晚推进，雨声越来越厚，低频越来越多。

### 2. 前台室内底噪

```text
office_hum_loop.wav
```

内容包括：

```text
荧光灯电流
旧冰箱/电箱低频
远处管道水声
偶尔的木头收缩声
```

这个声音很重要，它会让玩家感觉自己真的坐在旧公寓前台。

### 3. 门外空间

```text
outside_lobby_loop.wav
```

可以很轻，包括：

```text
雨打玻璃
远处车辆
门口积水
偶尔楼道回音
```

当危险来客出现时，门外空间声音可以突然变干、变空。

## B. UI 操作音

这些音效决定“手感”。

```text

ui_click_paper.wav
ui_tab_switch.wav
ui_confirm.wav
ui_warning.wav
ui_error.wav
```

不要用太现代的游戏 UI 音。
尽量像真实物件：

```text
纸片移动
木质抽屉
金属按钮
旧塑料按键
档案夹扣子
```

## C. 文档与证据音

这是你游戏最核心的声音层。

### 文件打开

```text
doc_id_open.wav          打开证件
doc_profile_open.wav     打开住户档案
doc_rules_open.wav       打开通知规则
doc_log_open.wav         打开登记簿
```

四种文件的声音应该不同。

```text
证件：薄塑料卡片声
档案：厚纸翻页声
规则：公告纸摩擦声
登记簿：厚重旧书翻页声
```

### 保存证据

```text
evidence_pin.wav
evidence_mark_positive.wav
evidence_mark_conflict.wav
```

建议区分两种证据：

```text
正向证据：清脆“咔”
异常证据：低一点、带摩擦的“咔”
```

玩家会逐渐形成条件反射。

---

## D. 电话系统音效

电话是你的恐怖核心之一。
它不应该只是工具，而是“另一个空间的入口”。

### 基础电话音

```text
phone_pickup.wav
phone_dial.wav
phone_ring_loop.wav
phone_connect.wav
phone_hangup.wav
phone_dead_air.wav
```

### 电话结果音

```text
phone_confirm_normal.wav      正常确认
phone_no_answer.wav           无人接听
phone_busy.wav                忙线
phone_duplicate_voice.wav     电话和门外同声
phone_fake_confirm.wav        伪造确认
phone_security_warning.wav    安保警告
```

最重要的是 `phone_duplicate_voice.wav`。

比如第三夜规则里有“同一声音同时出现在电话与门外，立即呼叫安保”。这个声音应该非常有记忆点：

```text
电话里说话
门外的人也同步说同一句
两边声音有 0.2 秒延迟
然后电话里出现轻微回音
```

这个会非常恐怖，而且是玩法证据。

---

## E. 扫描器音效

扫描器是“系统是否可信”的象征。
你的世界观里外部系统也会说谎，所以扫描器不能永远像可靠机器。

### 基础扫描

```text
scanner_start.wav
scanner_processing_loop.wav
scanner_pass.wav
scanner_fail.wav
scanner_timestamp_error.wav
scanner_company_error.wav
scanner_fake_pass.wav
scanner_jam.wav
```

### 关键设计

普通通过：

```text
短 beep + 干净收尾
```

异常失败：

```text
beep 断掉 + 电流噪音
```

伪通过：

```text
听起来像通过，但尾音多一个很轻的反向 beep
```

第六夜以后，玩家如果太依赖扫描器，复制体会伪造扫描结果。这个时候 `scanner_fake_pass.wav` 就很重要。它让老玩家感觉：

> 等等，这个通过声不太对。

---

## F. CCTV 音效

CCTV 要有“旧监控”的触感。

```text
cctv_power_on.wav
cctv_channel_switch.wav
cctv_static_loop.wav
cctv_freeze.wav
cctv_snapshot.wav
cctv_glitch_short.wav
cctv_shadow_missing.wav
cctv_impossible_floor.wav
```

操作设计：

```text
切换频道：咔 + 电流跳变
冻结画面：磁带卡住声
保存截图：老式相机/打印机声
异常帧：一瞬间低频抽动
```

第七夜最终案需要用“前台自身 CCTV”证明当前门岗仍在岗。这个地方音效应该特殊：

```text
cctv_self_view.wav
```

当玩家看到自己还坐在前台时，声音可以突然变得很近，像监控正在看玩家本人。

---

## G. 提问系统音效

提问不是普通对话，它是审讯。

```text
question_select.wav
question_ask.wav
visitor_answer_normal.wav
visitor_answer_nervous.wav
visitor_answer_wrong.wav
visitor_answer_too_perfect.wav
visitor_silence.wav
```

你的第六夜有“过度完美回答本身就是异常”。这个可以通过声音做出来：

正常回答：

```text
有自然停顿，有呼吸
```

复制体完美回答：

```text
没有呼吸
语速太稳
尾音像录音回放
```

如果没有配音，也可以用抽象声音表示：

```text
正常回答：低声 murmuring + 纸面文字出现
异常回答：murmuring 轻微倒放 + 文字出现
```

# 四、四种裁决的声音设计

这是最关键的。

## 1. Allow entry / 允许进入

感觉：正确、放行、门锁打开，但仍有一点不安。

```text
stamp_allow.wav
door_unlock.wav
visitor_steps_inside.wav
lobby_reverb_fade.wav
```

如果是真住户：

```text
声音干净，脚步自然远去
```

如果错放复制体：

```text
门锁打开后，脚步声突然变成两组
或者远处传来纸张被撕掉的声音
```

---

## 2. Refuse entry / 拒绝进入

感觉：关闭、阻止、雨夜里的沉默。

```text
stamp_reject.wav
door_bolt_lock.wav
visitor_steps_away.wav
rain_swell_short.wav
```

如果拒绝正确：

```text
门外脚步离开，雨声恢复
```

如果错拒真人：

```text
脚步没有离开
门外轻敲一次
电话突然响一下
```

这样玩家会知道自己可能做错了。

---

## 3. Call security / 呼叫安保

感觉：危险、强制处理、系统介入。

```text
stamp_security.wav
security_alarm_short.wav
radio_static.wav
metal_gate_lock.wav
```

如果正确：

```text
对讲机里短促回应
门外人影消失
```

如果误报：

```text
对讲机沉默
声誉值受损音
远处住户抱怨声
```

---

## 4. Hold / 留置等待

感觉：时间暂停，现实暂时没被写入。

```text
stamp_wait.wav
clock_tick_loud.wav
paper_ink_slow.wav
visitor_breath_close.wav
```

留置后进入调查分支：

```text
等待回拨：电话等待音
复查监控：CCTV 噪点增强
观察反应：呼吸声/衣料声/脚步不安
```

留置不应该听起来像“跳过”。
它应该听起来像把对方暂时钉在现实边缘。

# 五、夜晚递进音效设计

七夜每一夜的声音应该越来越不可靠。

## 第一夜：正常办公恐怖

```text
雨声轻
电话正常
扫描器正常
印章很清楚
```

重点是教学。

## 第二夜：外部系统开始说谎

```text
电话偶尔有杂音
扫描器开始出现轻微延迟
蓝星维修相关案件出现远处车辆声
```

## 第三夜：同一个人出现在两处

```text
电话回音变多
房间线路声音和门外声音重叠
CCTV 有单帧异常
```

## 第四夜：私人习惯

```text
来客个人声音标签变重要
手杖、琴盒、花束、舞蹈包这些声音更明显
```

## 第五夜：档案污染

```text
翻页声开始不稳定
登记簿自己写字
纸张有湿墨声
档案打开时有低频
```

## 第六夜：它们学会了你的流程

```text
工具反馈变得“过度干净”
伪通过声音出现
玩家常用工具的声音开始不可信
```

这是很强的设计点：
**工具音效也被复制体学习了。**

## 第七夜：等待替换

```text
环境音变空
雨声像从室内传来
电话里偶尔传来自己的呼吸
印章声变得特别重
最终复制体出现时，所有 UI 音短暂静音
```

最终案“第二个薛夜”出现时，建议直接做一次声音断层：

```text
所有底噪停止 0.5 秒
门外敲门声响起
值班铃自己响一下
电话轻微接通，但没人说话
```

这个会非常有压迫感。

# 七、音效如何参与教程

第一案可以这样用声音引导玩家：

```text
1. 证件页轻轻闪烁，同时播放 doc_hint.wav
2. 玩家点证件，播放 doc_id_open.wav
3. 档案页亮起，播放 soft_tick.wav
4. 玩家发现姓名冲突，播放 evidence_hint.wav
5. 保存证据，播放 evidence_save_conflict.wav
6. 拒绝章亮起，低频消失
7. 玩家盖章，播放 stamp_reject.wav
8. 门外脚步离开，玩家知道做对了
```

这样玩家不用读很多教学文字，也知道下一步该做什么。

# 八、混音规则

这个游戏声音多，如果不控制，会乱。

建议音量层级：

```text
环境底噪：20% - 35%
UI 点击：35% - 45%
工具反馈：45% - 60%
印章裁决：70% - 85%
警报/重大异常：85% - 100%
```

不要让雨声盖过工具声音。
不要让 UI 声太尖。
印章声一定要厚、近、重。

# 十、最终建议：声音要做成“核验的一部分”

你这个游戏最好的音效方向不是单纯恐怖，而是：

```text
玩家通过声音也能判断异常。
```

比如：

```text
真住户敲门节奏稳定
复制体敲门节奏像模仿出来的

真电话有房间环境底噪
假电话太干净

真扫描通过声短促
伪扫描通过声尾音反向

真 CCTV 冻结有磁带卡顿
假 CCTV 冻结太顺滑

正确盖章后环境恢复
错误盖章后雨声或纸声变得不自然
```

这样音效就不只是装饰，而是玩法信息。

你下一步可以先做一个 **v4 音效最小闭环**：

```text
雨声循环
文件翻页
电话三件套
扫描通过/失败
CCTV 切换/冻结
保存证据
四种印章
正确/错误反馈
门锁打开/锁住
安保警报
```

只要这套做好，《午夜登记簿》的互动感会明显提升一大截。
对，你现在应该做的不是“加几张恐怖怪物图”，而是把《午夜登记簿》的敌人系统升级成一个完整玩法层。

你现在已有七夜、68 个案件、电话/扫描/CCTV/提问/留置/裁决/结局这些基础系统，所以最好的改法不是推倒重来，而是把“伪人核验”升级成：

> **门岗身份核验 + 生物畸变暴露 + 异常处理决策**

这样既保留你原来的核心，又能加入《生化危机》《最后生还者》那种强烈怪物压迫感。当前项目已经有七夜结构和完整核验工具链，这正好适合加一层“怪物暴露与处置系统”。

---

# 《午夜登记簿》v4 怪物系统升级计划

## 一、v4 核心目标

v4 的目标不是把游戏改成打丧尸，而是让玩家感受到：

> 门外一开始是“人”。
> 查着查着，玩家发现它不对劲。
> 继续逼问或核验后，它开始失去人形。
> 最后玩家必须判断：这是什么类型的异常？该怎么处理？

所以 v4 的核心体验应该变成：

```text
来客到达
↓
身份核验
↓
发现破绽
↓
局部畸变暴露
↓
判断怪物类型
↓
选择处理方式
↓
决定后果
```

这个比单纯“看证件找错字”强很多。

---

# 二、世界观重写方向

现在的“伪人”可以升级成：

## 敌人总称：借壳者

它们不是普通丧尸，也不是纯鬼怪。
它们是一种会借用人类身份、外貌和生活记录的畸变体。

它们的目标不是吃人，而是：

> 通过登记簿获得“合法存在权”。

一旦玩家盖章允许，它们就能进入楼内，占用某个真实住户的身份。真实住户可能会被替换、遗忘、困在楼外，甚至从档案中消失。

但它们有一个弱点：

> 它们的身份可以伪造，肉体却不稳定。
> 被识破后，伪装层会崩解。

这样你就能保留原来的登记簿主题，同时加入生物恐怖。

---

# 三、怪物设计不要只有一种

你不要把所有敌人都做成“丧尸脸裂开”。那样很快会腻。

你应该设计 6 种怪物类型，每种有不同外观、破绽、暴露方式和处理方法。

---

## 1. 模仿失败型

### 玩法定位

最基础的伪装怪。
适合第一夜到第三夜出现。

### 初始状态

看起来像普通住户，但细节不对：

```text
五官比例轻微错误
笑容停留太久
眨眼太少
说话像背台词
站姿过于端正
```

### 暴露方式

被问到私人习惯时，它会卡住。

例如：

```text
玩家问：你平时敲门是几下？
它回答：三下。
玩家继续问：那你为什么刚才敲了两下？
它沉默。
嘴角开始抽动。
脸部像没贴好的面具一样松开。
```

### 完全暴露形态

```text
面皮裂开
下面还有一张没完成的人脸
嘴巴横向拉长
手指像被重新排列
身体试图重新摆成人类姿势
```

### 正确处理

```text
拒绝进入
或
呼叫安保
```

### 错误后果

放行后，它会替换低风险住户。

---

## 2. 寄生增殖型

### 玩法定位

生物恐怖核心类型。
适合第四夜以后出现。

### 初始状态

看起来像人，但身体有感染迹象：

```text
颈部有轻微鼓动
耳后有暗色斑点
衣领下有湿痕
呼吸声不自然
皮肤下面像有东西在移动
```

### 暴露方式

扫描或强光照射后开始反应。

```text
扫描器扫过胸口时出现异常热区
CCTV 里脖子后面有东西开合
电话确认本人在楼上后，它开始剧烈呼吸
```

### 完全暴露形态

```text
头部或胸腔像花一样裂开
喉咙变成孢子喷口
肩背长出菌丝状结构
说话不再用嘴，而是胸腔共振
```

### 正确处理

新增处理方式：

```text
封锁入口
净化喷雾
呼叫安保
```

### 错误后果

不是立刻 game over，而是：

```text
大厅污染值上升
后续来客更容易出现感染迹象
某些 CCTV 频道出现菌丝遮挡
```

---

## 3. 骨架错构型

### 玩法定位

强视觉冲击型。
适合做中期恐怖高潮。

### 初始状态

远看像人，近看肢体比例不对：

```text
手臂过长
肩膀高度不一致
膝盖方向不自然
站着像在学习站立
敲门动作像昆虫腿模仿人
```

### 暴露方式

CCTV 或观察记录最容易发现。

```text
正面看正常
走廊摄像头里它的腿是反折的
玻璃倒影里它的脖子比现实更长
```

### 完全暴露形态

```text
四肢反向展开
脊柱拱起
头歪到不可能的角度
整个人趴在玻璃上
手掌裂成抓握器官
```

### 正确处理

```text
拉下铁门
封锁入口
呼叫安保
```

### 错误后果

它会攻击门禁结构，导致后续几案“门锁稳定度下降”。

---

## 4. 空壳回声型

### 玩法定位

高级精神恐怖。
适合第五夜、第七夜。

### 初始状态

它不像感染者，更像“里面没有人”。

```text
表情太淡
声音像录音
动作有轻微卡帧
眼睛反光不对
CCTV 里脸部细节缺失
```

### 暴露方式

电话和声音最容易识破。

```text
电话里的声音和门外声音完全同步
玩家挂断电话后，门外还在继续说电话里的最后一句
```

### 完全暴露形态

```text
嘴里没有喉咙，是黑色空洞
眼眶里没有眼球
身体内部像空壳
被强光照到时边缘缺失
```

### 正确处理

```text
强光照射
广播干扰
拒绝进入
```

### 错误后果

它不会立刻杀人，而是污染档案：

```text
住户照片变空白
电话线路出现回声
登记簿自动补写不存在的人名
```

---

## 5. 拼接学习型

### 玩法定位

第六夜核心怪。
它对应你原来“它们学会了你的流程”的设定。

### 初始状态

非常完美，几乎没有破绽。

```text
证件正确
回答正确
扫描通过
电话确认也正常
```

但它的问题是：

```text
太完美
没有自然犹豫
回答像从数据库拼出来
会使用多个住户的私人习惯
```

### 暴露方式

必须用交叉提问逼它出错。

例如：

```text
它知道林安娜的敲门节奏
也知道孙浩会带包子
还知道王玉兰的手杖
但这些习惯不该属于同一个人
```

### 完全暴露形态

```text
身体上拼接多个住户特征
一只手戴周启明的手套
肩上有林安娜舞蹈包带痕
嘴里说出不同住户的口头禅
脸部快速切换成几个人的表情
```

### 正确处理

```text
留置诱导
连续提问
保存三条矛盾证据
呼叫安保
```

### 错误后果

它会学习玩家本局最常用工具，让后面工具结果更不可信。

---

## 6. 前台替换型

### 玩法定位

最终 Boss。
第七夜最终来客。

### 初始状态

它不是怪物外表，而是另一个“薛夜”。

```text
证件完整
工牌正确
知道值班流程
知道玩家之前做过的裁决
甚至知道某些玩家选择
```

### 破绽

```text
今晚没有换班
000 室不是有效房间
当前前台 CCTV 已经拍到玩家在岗
出入登记簿已经记录薛夜正在值班
它没有倒影
它的影子比人慢半秒
```

### 暴露方式

当玩家保存足够证据并呼叫安保后，它才放弃伪装。

### 完全暴露形态

```text
脸仍然是薛夜
但身体内部是空壳和拼接组织
胸口像登记簿纸页一样开合
皮肤下面浮现玩家之前盖过的印章
它的声音开始重复玩家做过的错误裁决
```

### 正确处理

```text
呼叫安保
封锁入口
使用前台 CCTV 证明当前自己已登记
使用登记簿反写“不得登记第二个自己”
```

### 错误后果

坏结局：

```text
新的薛夜坐在前台
真正的薛夜站在雨中敲玻璃
```

---

# 四、v4 新增玩法系统

你需要增加一个新系统：**暴露等级系统**。

每个异常来客都有一个 `exposureLevel`。

```ts
exposureLevel: 0 | 1 | 2 | 3;
```

对应：

```text
0 = 完全伪装
1 = 轻微异常
2 = 局部畸变
3 = 完全暴露
```

---

## 暴露等级如何上升？

玩家做以下行为会推进暴露：

```text
发现关键矛盾
保存正确证据
使用对应工具
问中陷阱问题
留置等待
CCTV 捕捉异常帧
电话确认本人已在楼内
```

例如：

```text
模仿失败型：
私人提问命中 → exposure +1

寄生增殖型：
扫描异常热区 → exposure +1
强光照射 → exposure +1

空壳回声型：
电话同步声音 → exposure +1
广播干扰 → exposure +1

骨架错构型：
CCTV 侧面视角 → exposure +1
```

---

# 五、裁决系统也要升级

现在你的裁决是：

```text
允许
拒绝
呼叫安保
留置等待
```

v4 可以保留这四个，但在怪物暴露后增加“异常处理动作”。

新增处理动作建议：

```text
封锁入口
强光照射
广播干扰
净化喷雾
拉下铁门
登记簿反写
```

不要一开始全开放。按夜晚逐步解锁。

---

## 每种怪物对应处理方式

| 怪物类型  | 最有效处理                |
| ----- | -------------------- |
| 模仿失败型 | 拒绝 / 安保              |
| 寄生增殖型 | 净化喷雾 / 封锁            |
| 骨架错构型 | 拉下铁门 / 安保            |
| 空壳回声型 | 强光 / 广播干扰            |
| 拼接学习型 | 留置诱导 / 多证据安保         |
| 前台替换型 | CCTV 自证 / 登记簿反写 / 安保 |

这样玩家不是永远点“呼叫安保”，而是要判断敌人类型。

---

# 六、七夜结构怎么改

你原来的七夜结构可以保留，但每夜加入新的怪物层。

---

## 第一夜：人不像人

目标：教玩家基础核验和轻微暴露。

新增重点：

```text
模仿失败型
轻微脸部抽动
错误笑容
名字和外貌破绽
```

第一夜不要太怪。
只让玩家看到“这东西不完全是人”。

结尾事件：

```text
被拒绝的伪人离开前，脸上的笑容慢半拍才消失。
```

---

## 第二夜：外部系统也被感染

目标：让玩家知道公司、工单、扫描器也会骗人。

新增重点：

```text
寄生增殖型初登场
蓝星维修车内部有湿痕
工牌背面有生物组织
```

第二夜第一个小高潮：

```text
玩家扫描维修工证件。
扫描通过。
但 CCTV 热成像显示他胸口有第二个呼吸源。
```

---

## 第三夜：同一个人出现在两处

目标：电话恐怖。

新增重点：

```text
空壳回声型
电话与门外同步说话
房间里的人还活着，但门外也有一个“他”
```

第三夜强场面：

```text
玩家拨打 208。
电话里朴米娜说：“我在楼上。”
门外朴米娜同时也说：“我在楼上。”
然后门外的人停止眨眼。
```

---

## 第四夜：私人习惯与肉体习惯

目标：私人习惯不仅是文本，也是动作和声音。

新增重点：

```text
骨架错构型
人物动作模仿失败
敲门节奏、走路姿势、拿物方式成为证据
```

第四夜让玩家学会：

> 不是只看它说什么，也要看它怎么动。

---

## 第五夜：档案污染变成生物污染

目标：档案室开始长出东西。

新增重点：

```text
档案污染
寄生体污染纸张
登记簿页面出现肉膜/湿墨
```

第五夜可以增加一个机制：

```text
污染值
```

错放寄生体会提高污染值。
污染值高时，档案页会更难读，CCTV 更容易花屏。

---

## 第六夜：它们学会了你

目标：拼接学习型登场。

新增重点：

```text
怪物根据玩家常用工具伪装
伪造扫描通过音
伪造电话确认
模仿玩家之前接受过的答案
```

第六夜核心规则：

> 越像正确答案，越可能是陷阱。

---

## 第七夜：第二个前台

目标：最终替换危机。

新增重点：

```text
前台替换型
它不是来冒充住户
它是来冒充玩家
```

最终案不要只是判断真假。
应该是完整流程：

```text
查看它的换班文件
发现 000 工号
查今晚规则：没有换班
切前台 CCTV：玩家仍在岗
查登记簿：薛夜已登记为当前门岗
强光照射：它没有倒影
保存证据
登记簿反写
呼叫安保
封锁入口
```

---

# 八、UI 需要怎么改

v4 必须让玩家看到“怪物正在暴露”。

你需要新增一个视觉模块：

## 异常观察面板

放在来客头像旁边。

```text
异常观察

[ ] 呼吸节奏
[ ] 手部结构
[ ] 眼神焦点
[ ] 声音同步
[ ] 影子/倒影
[ ] 皮肤表面
```

玩家使用工具后，这些项目会逐渐揭示。

---

## 暴露等级条

不要叫“血条”，可以叫：

```text
伪装稳定度
```

显示方式：

```text
伪装稳定度：稳定 → 波动 → 崩解 → 暴露
```

越被识破，伪装越崩。

但注意：
不是所有怪物都必须暴露到最后。
有时候玩家在轻微异常阶段就可以拒绝。

---

## 怪物分类标签

当玩家收集足够证据后，可以让系统显示推测：

```text
疑似类型：空壳回声型
推荐处理：强光 / 安保
```

前期不要直接告诉玩家，后期可以通过升级解锁。

# 九、素材计划

你需要的不是每个怪物都画完整动画。
为了省成本，可以做“三阶段素材”。

每个高危怪物准备：

```text
normal.png      正常伪装
reveal_1.png    局部异常
reveal_2.png    明显畸变

```

第一批只做 6 个怪物模板：

```text
mimic_failed_normal/exposed
parasite_bloom_normal/exposed
structure_breaker_normal/exposed
hollow_echo_normal/exposed
adaptive_collector_normal/exposed
frontdesk_replacement_normal/exposed
```

不用 68 个全画。
先做模板，再在案件里复用。

不需要一开始做复杂动画。

```text

shadow_delay       影子延迟
limb_extend        肢体拉长
skin_bloom         皮肤开花
hollow_flicker     空壳闪烁
```

## 触发时机

```text

```

这样怪物不是静态图，而是跟玩家操作绑定。

# 十二、程序结构建议

你可以新增这些字段到 `Visitor`。

```ts
type MonsterType =
  | "none"
  | "failed_mimic"
  | "parasite_bloom"
  | "structure_breaker"
  | "hollow_echo"
  | "adaptive_collector"
  | "frontdesk_replacement";

type ExposureStage = 0 | 1 | 2 | 3;

type ContainmentAction =
  | "reject"
  | "security"
  | "lockdown"
  | "uv_light"
  | "broadcast"
  | "cleanse"
  | "registry_rewrite";
```

然后每个怪物案件增加：

```ts
monsterProfile: {
  type: "parasite_bloom",
  exposureStage: 0,
  revealTriggers: ["scanner_heat", "cctv_neck_pulse"],
  correctContainment: ["lockdown", "cleanse"],
  wrongContainmentEffect: "lobby_contamination_up"
}
```

你还需要一个纯函数：

```ts
getExposureUpdate({
  visitor,
  action,
  selectedEvidence,
  toolResult
});
```

还有一个处理函数：

```ts
resolveContainment({
  visitor,
  decision,
  containmentAction,
  exposureStage,
  evidenceCount
});
```

这样以后怪物逻辑不会全部塞进主组件。

# 十四、最重要的设计原则

你一定要记住这 5 条。

## 1. 怪物必须先像人

不要一上来就是怪物。
恐怖来自：

> 它差一点像人，但又不完全像人。

---

## 2. 暴露必须由玩家触发

不要只是时间到了变身。
应该是玩家查到了关键证据，它才崩。

这样玩家会有成就感。

---

## 3. 每种怪物要有对应处理方式

否则所有案件最后都会变成“呼叫安保”。

玩家应该思考：

```text
这是寄生型？要净化。
这是空壳型？要强光。
这是骨架型？要封锁。
这是学习型？要留置诱导。
```

---

## 4. 不要全员都是怪物

真人必须存在。
而且真人要更有生活感。

否则玩家会默认“全拒绝”，游戏就没意思了。

---

## 5. 怪物越恐怖，证据链越重要

不能让玩家靠外观猜。
要让玩家通过操作一步步揭开它。

```text
证件
档案
电话
CCTV
提问
观察
留置
暴露
处理
```

这才是你的核心循环。

---

# 十五、最终一句话定位

你可以把 v4 定义成：

> 《午夜登记簿》是一款门岗身份核验生物恐怖游戏。玩家隔着玻璃审查夜访者，通过证件、电话、监控、提问和私人记忆识别借壳者。当伪装被揭穿，它们会暴露出不同的畸变形态，玩家必须选择正确的封锁、强光、净化、安保或登记簿处理方式，阻止它们被现实承认为“人”。

这个方向会比单纯“伪人找不同”更强，也不会丢掉你原本最有特色的门岗玩法。
