# 《午夜登记簿》V4 游戏体验提升计划书

> 实施状态（2026-06-14）：全部 Sprint、准备台素材、音频、动画、剧情修订、验证器和调试页已完成。最终实现与复审记录见 `docs/midnight_registry_animation_story_audit.md` 第 13 节。

## 一、当前问题诊断

### 1. 最大问题

当前版本已经具备很多玩法系统，但玩家体验仍然像在阅读文字和管理面板，而不是在进行一个紧张的身份核验游戏。

主要原因不是内容少，而是：

1. 玩家一打开不知道目标。
2. 首局没有明确教程引导。
3. 信息一次性展示太多，玩家不知道先看哪里。
4. 互动工具虽然多，但很多结果仍以文字为主。
5. 玩家缺少“我正在破案”的过程反馈。
6. 七夜剧情有设定，但每一夜的即时目标不够清楚。
7. 部分动画存在，但没有完全变成玩家操作后的强反馈。

### 2. 当前体验像什么

现在更像：

“我打开一个复杂的档案系统，读规则、读文档、读电话结果，然后选一个按钮。”

目标体验应该是：

“我坐在午夜门岗室里，一个人敲窗，我必须亲手查证件、打电话、看监控、问问题、标记证据，最后决定要不要让这个东西进来。”

---

## 二、V4 改进总目标

### 核心目标

把《午夜登记簿》从“文字型系统 Demo”提升为：

**有明确目标、有强教程、有高互动密度、有视觉反馈、有剧情推进的夜班门岗身份核验恐怖游戏。**

### 玩家第一分钟必须理解的事情

玩家进入游戏后，必须在 60 秒内知道：

1. 我是月影公寓的夜班门岗。
2. 我的任务是处理今晚 8 位来客。
3. 我必须保护三条数值：安全、声誉、稳定度。
4. 每个来客都要通过证件、档案、电话、CCTV、提问和证据判断。
5. 错误放行伪人会让住户被替换。
6. 错拒真人会降低声誉。
7. 呼叫安保必须有足够证据。
8. 留置可以争取更多线索，但会增加压力。

### V4 成功标准

V4 不是增加更多案例，而是让前 10 分钟变得清楚、紧张、有手感。

最低成功标准：

1. 新玩家能不看说明完成第一夜前 3 个案例。
2. 第一位来客有完整新手引导。
3. 玩家每 5 到 10 秒都有一个可执行动作。
4. 每个工具都有明确动画和反馈。
5. 每个案例都有“证据进度”或“当前目标”提示。
6. 玩家知道为什么自己赢或输。
7. 前三夜不剧透第七夜最终复制体。
8. CCTV 后 4 个素材在第五至第七夜进入实际玩法。

## 三、第一阶段：重做开局体验

### 目标

解决玩家一打开不知道该干嘛的问题。

### 当前问题

现在开局是 Daytime Prep Desk，包含模式选择、规则、perk 和 Start Night Shift。结构是对的，但新玩家会先看到很多系统选择，却还不知道核心玩法是什么。

### 改进方案

开局页面应该先回答三个问题：

1. 你是谁？
2. 今晚目标是什么？
3. 怎么开始第一班？

### 新开局结构

建议改成以下顺序：

#### 1. 简短身份说明

页面顶部显示一句话：

“你是月影公寓的新任午夜门岗。今晚有 8 位来客。你的工作是把真正的人放进来，把伪人挡在门外。”

#### 2. 今晚目标卡

显示清晰目标：

* 处理 8 位来客
* 安全值不要归零
* 声誉值不要归零
* 稳定度不要归零
* 错放伪人会降低安全
* 错拒真人会降低声誉
* 异常事件会降低稳定度

#### 3. 默认推荐模式

新玩家只突出一个按钮：

“开始故事模式”

Challenge 和 Endless 不要和故事模式并列抢注意力。可以放在下面较小区域，写成：

“完成第一夜后推荐尝试。”

#### 4. Perk 简化

第一夜不要让玩家选择太多 perk。第一夜可以只给一个固定 perk：

“值班手册：第一夜会提示你检查顺序。”

第二夜开始再开放 Sanity Coffee、ID Laser Booster、Manual Lock Override、Switchboard Kit。

### 执行任务

1. 重排 Daytime Prep 页面。
2. 把模式选择从三个大按钮改为“推荐故事模式 + 可选高级模式”。
3. 第一夜默认锁定 Tutorial Perk。
4. Start Night Shift 改成“开始第一夜值班”。
5. 点击开始后播放 Night Transition。

### 验收标准

新玩家打开页面后，不需要滚动、不需要读长文，就能看懂今晚目标。

## 四、第二阶段：加入“当前目标”系统

### 目标

让玩家每一刻都知道下一步该做什么。

### 当前问题

玩家看到很多面板：访客、文件、档案、工具、规则、CCTV、电话、证据、决策。系统很丰富，但缺少一个“导演”告诉玩家现在应该关注什么。

### 新增系统：Current Objective Panel

在主游戏界面顶部或左上角固定显示当前目标。

例子：

#### 第一位来客

“目标 1 / 6：查看来客声明。”

完成后变成：

“目标 2 / 6：打开住户档案，核对姓名和房号。”

再变成：

“目标 3 / 6：点击照片备注，检查外貌特征。”

再变成：

“目标 4 / 6：标记至少 1 条证据。”

再变成：

“目标 5 / 6：选择处理方式。”

最后：

“目标 6 / 6：查看结果。”

### 目标类型

每个案例都应该有一个动态目标：

1. 查看来客声明
2. 查看证件
3. 查看档案
4. 查看今日规则
5. 使用一个验证工具
6. 标记证据
7. 做出裁决

### 不同夜晚的目标变化

第一夜目标更直接：

“检查姓名、房号、照片。”

第三夜目标变成：

“至少使用一次电话验证。”

第五夜目标变成：

“不要只相信档案，使用第二来源确认。”

第六夜目标变成：

“避免重复使用同一个工具组合。”

第七夜目标变成：

“证明当前门岗已经在岗。”

### 执行任务

1. 新增 currentObjective state。
2. 每个案例配置 recommendedSteps。
3. 玩家完成动作后推进 objective。
4. 如果玩家卡住 15 秒，显示轻提示。
5. 如果玩家卡住 30 秒，显示明确提示。

### 验收标准

玩家任何时候都能看到“我下一步应该做什么”。

## 五、第三阶段：重做第一夜教程

### 目标

让第一夜成为自然教学，而不是让玩家自己摸索。

### 教程原则

不要用一大段教学文字。
要用“边玩边教”。

第一夜前 3 个案例应该分别教会：

1. 案例 1：怎么查证件和档案。
2. 案例 2：怎么检查外貌特征。
3. 案例 3：怎么标记证据和做裁决。

### 第一位来客：超强引导案例

#### 来客设定

来客声称是林安娜，但名字或房号明显错误。

#### 教学流程

1. 来客敲窗。
2. 系统提示：“先查看她递来的证件。”
3. 高亮 Resident ID。
4. 玩家点击后，证件滑入桌面。
5. 系统提示：“现在打开住户档案。”
6. 高亮 Archive。
7. 玩家点击档案。
8. 系统提示：“对比姓名和房号。”
9. 错误字段轻微呼吸高亮，但不要直接说答案。
10. 玩家点击错误字段。
11. 系统提示：“这是一条证据，把它盖到证据板上。”
12. 高亮 Evidence Reason。
13. 玩家选择“姓名/ID 不符”。
14. 系统提示：“证据足够。拒绝进入。”
15. 玩家点击 Refuse。
16. 播放拒绝动画。
17. 结算弹窗解释：“正确。她的证件姓名与档案不符。”

### 第二位来客：外貌教学

教玩家看照片备注、痣的位置、手套左右。

### 第三位来客：安保教学

让玩家遇到一个危险伪人，教会：

“拒绝”和“呼叫安保”不是同一个东西。
普通不符可以拒绝。
复制声音、无影子、身份覆盖风险必须呼叫安保。

### 第一夜不应该教太多

第一夜暂时不要强行教：

* 故障小游戏
* 白日升级
* 学习型伪人
* 档案污染
* 多频道 CCTV
* 复杂关系网

这些留到后面。

### 执行任务

1. 第一夜第 1 案设置为强制引导案例。
2. 第一夜第 2 案设置为外貌观察案例。
3. 第一夜第 3 案设置为证据和裁决案例。
4. 前 3 案可以限制部分按钮，按教学顺序解锁。
5. 第一夜第 4 案开始允许玩家自由操作。

### 验收标准

新玩家完成第一夜前 3 案后，知道基本循环：

查看文件 → 查档案 → 找矛盾 → 标记证据 → 做裁决。

## 六、第四阶段：把“读文字”改成“做动作”

### 目标

减少纯阅读，把关键信息变成可点击、可比较、可标记、可反馈的互动。

### 当前问题

很多线索通过文字结果直接告诉玩家，例如电话结果、扫描结果、CCTV 结果。玩家是在读系统返回，而不是主动发现。

### 改进方向

所有关键线索都应该尽量变成以下形式：

1. 可点击字段
2. 可高亮差异
3. 可拖入证据板
4. 可截图或冻结
5. 可盖章确认
6. 可产生视觉反馈

---

### 1. 文件详情放大层

当前文件打开和关闭未完整接入。建议补齐。

#### 玩法

玩家点击证件后，打开一个文件详情层。

详情层中字段可点击：

* 姓名
* 房号
* ID
* 照片
* 日期
* 公司名
* 盖章
* 签名

玩家点击字段后，可以选择：

“加入证据板”

或者：

“与档案比较”

#### 例子

玩家点击证件姓名：

“Lin Anya”

再点击档案姓名：

“Lin Anna”

系统显示：

“发现可疑差异：姓名不符。”

但不要直接说真假，只提示这是证据。

#### 执行任务

1. 新增 DocumentModal。
2. 文件字段结构化，不只是一段文字。
3. 字段支持 click。
4. 字段支持 compareWithArchive。
5. 支持打开和关闭动画。
6. 支持证据盖章动画。

---

### 2. 证据板升级

当前证据标记已有，但还可以更像推理游戏。

#### 新机制

每个案例显示：

“证据完整度：0 / 2”

或：

“建议至少找到 2 条证据再呼叫安保。”

注意：这不是告诉玩家答案，只是告诉玩家证据够不够。

#### 证据类型

* 文件证据
* 外貌证据
* 电话证据
* CCTV 证据
* 行为证据
* 规则证据
* 出入记录证据

#### 执行任务

1. 每个案例配置 requiredEvidenceCount。
2. 证据板显示当前数量。
3. 呼叫安保前，如果证据不足，提示：“安保可能不会采信。”
4. 正确证据加入时播放 pin/stamp。
5. 错误证据加入时不要立刻惩罚，但结算解释。

---

### 3. CCTV 定格和截图

当前 CCTV 有频道切换，但缺少“我发现了东西”的互动。

#### 新玩法

玩家打开 CCTV 后，可以：

1. 切换频道。
2. 点击画面冻结。
3. 点击可疑区域。
4. 保存截图为证据。

#### 例子

电梯频道显示：

“楼层：7”

玩家点击楼层数字，系统保存：

“CCTV 证据：电梯显示不存在楼层。”

#### 执行任务

1. 给 CCTV 图片增加 hotspot 区域。
2. 点击 hotspot 触发 freeze zoom。
3. 点击“保存为证据”。
4. 接入 CCTV 放大定格动画。
5. 第五至第七夜解锁后 4 个 CCTV 频道。

---

### 4. 电话系统升级

当前电话结果偏文字。要让电话成为一个小流程。

#### 电话流程

点击电话后：

1. 电话拨号盘出现。
2. 玩家选择拨打对象。
3. 拨号动画 0.8 秒。
4. 返回状态：

* 接通
* 无人接听
* 死寂
* 忙音
* 重复声音
* 错误号码
* 管理处异常

#### 每种状态都有不同反馈

接通：绿色小灯亮。
无人接听：电话按钮变暗。
死寂：低频噪声、字幕延迟。
重复声音：屏幕轻微失真。
异常通话：Stability 下降。

#### 执行任务

1. 新增 PhonePanel。
2. 电话结果不要直接出现在 log 里。
3. 接入 phone dialing、connected、dead air、bad call distortion。
4. 电话结果可以保存为证据。
5. 异常电话立即触发视觉失真，而不是等错误裁决。

---

### 5. 扫描器反馈升级

当前扫描有扫描线，但缺少 pass/error 强反馈。

#### 新玩法

扫描不应该只是文字：

1. 点击扫描器。
2. 扫描线扫过证件。
3. 出现结果灯：

* ID Match
* Expired
* Format Error
* Registry Mismatch
* Corrupted Record

#### 执行任务

1. 补扫描通过动画。
2. 补扫描错误动画。
3. 扫描结果可以保存为证据。
4. 故障时不要只写文字，要显示红灯、错误码、重试按钮。

---

### 6. 提问卡加强

当前提问卡翻转太短，反馈不够强。

#### 新设计

玩家点击提问卡后：

1. 卡片翻面。
2. 来客进入 talking 状态。
3. 台词逐字出现。
4. 如果命中陷阱，卡片边缘红光。
5. 来客 mood 变 nervous / suspicious / angry。
6. 回答可保存为证据。

#### 执行任务

1. 接入 talking 状态。
2. 延长 question flip 动画至 0.45 秒。
3. 陷阱命中添加独立特效。
4. 回答内容显示在 Dialogue Box，而不是只在工具 log。
5. 提问结果进入 Evidence Board。

---

## 七、第五阶段：做一个真正的“案件流程”

### 目标

让每个来客像一个小案件，而不是一张资料卡。

### 每个案例的标准流程

每个来客应该有 6 个阶段：

1. Arrival：来客出现。
2. Claim：来客自述。
3. Documents：查看证件。
4. Verification：使用至少一个工具。
5. Evidence：标记证据。
6. Decision：做出裁决。

### UI 上显示案件阶段

在界面上加一个小的 Case Progress：

“Arrival → Documents → Verify → Evidence → Decision”

当前阶段高亮。

### 为什么重要

这可以立刻解决玩家“不知道干嘛”的问题。
玩家看到流程条，就知道自己还没完成哪一步。

### 不要强制所有阶段

第一夜前 3 案可以强制。
后面允许跳过，但会显示风险：

“你尚未标记证据，直接裁决可能导致误判。”

### 执行任务

1. 新增 casePhase。
2. 每个玩家操作推进 casePhase。
3. 决策按钮旁显示 Evidence Count。
4. 证据不足时给软提醒。
5. 案例结算解释玩家跳过了哪些验证步骤。

---

## 八、第六阶段：重新设计结算反馈

### 目标

让玩家知道自己为什么对、为什么错，而不是只看到结果。

### 当前问题

正确和错误结果有弹窗，但教育性不足。
玩家可能不知道自己哪里漏查。

### 新结算结构

每个案例结束后显示：

#### 1. 裁决结果

“正确处理”或“错误处理”。

#### 2. 真实身份

“真实住户”或“伪人”。

#### 3. 你找到的证据

列出玩家标记过的证据。

#### 4. 你漏掉的关键证据

最多显示 1 到 2 条，不要太多。

#### 5. 后果

* 安全变化
* 声誉变化
* 稳定度变化
* 住户状态变化

#### 6. 下一案提示

“下一位来客可能需要电话确认。”

### 执行任务

1. 重做 ResultModal。
2. 增加“你漏掉了什么”。
3. 增加 correct stamp animation。
4. 增加 reputation damage 和 safety damage 独立动画。
5. 结算文字控制在 5 行以内，避免又变成长文。

---

## 九、第七阶段：重做第三夜剧情，保护第七夜高潮

### 目标

修复第三夜提前剧透第七夜的问题。

### 当前问题

第三夜已经出现完整的薛夜复制体换班案例，导致第七夜最终冲突重复。

### 改进方案

第三夜不要让第二个薛夜实体站到门外。

第三夜可以改成三个较弱的铺垫版本之一。

---

### 方案 A：错误换班通知

第三夜最后一案不是“薛夜复制体来访”，而是收到一张换班通知：

“03:00 会有一名替班门岗到达。”

但电话管理处说：

“今晚没有换班。”

CCTV 中短暂出现一个穿门岗制服的人影，但人影没有敲门，马上消失。

正确处理：

“标记为异常通知，不允许登记。”

好处：

只埋伏笔，不完成高潮。

---

### 方案 B：电话里的第二个薛夜

第三夜电话响起。电话里有人说：

“我是薛夜，我在门外，为什么里面已经有人？”

但门外没有人。

正确处理：

“呼叫安保或记录异常通话。”

好处：

制造身份恐惧，但不让复制体正式登场。

---

### 方案 C：CCTV 一帧异常

CCTV 前台频道中出现一帧：

另一个薛夜坐在前台。

但玩家回到主界面时一切正常。

正确处理：

“保存 CCTV 证据，继续值班。”

好处：

非常恐怖，而且保留第七夜实体冲突。

---

### 推荐选择

推荐使用方案 A + 方案 C。

第三夜结尾流程：

1. 收到错误换班通知。
2. 管理处否认换班。
3. CCTV 一帧出现另一个前台。
4. 玩家保存异常证据。
5. 夜晚结算里显示：“你的工牌被查询了一次。”
6. 第七夜再让第二个薛夜正式敲窗。

### 执行任务

1. 删除或重写第三夜完整薛夜复制体案例。
2. 第三夜只保留“换班异常”和“前台影像”。
3. 第七夜最终案例强化为唯一实体复制体。
4. 统一第三夜正确处理方式。
5. 更新七夜剧情文档。

---

## 十、第八阶段：提升七夜 engagement 节奏

### 目标

让玩家每一夜都有新的期待，而不是只是更多案例。

### 每夜必须有三个东西

每一夜开始时给玩家：

1. 今夜目标
2. 今夜新规则
3. 今夜剧情悬念

### 改进版七夜结构

#### 第一夜：看懂身份

目标：学会基本核验。
新规则：证件和档案必须一致。
悬念：第一个伪人说出了正确问候，但证件错了。

#### 第二夜：外部系统会说谎

目标：学会查预约、公司和楼层规则。
新规则：维修和快递必须看预约。
悬念：蓝星维修十二年前已经注销。

#### 第三夜：同一个人出现在两处

目标：学会电话验证。
新规则：电话冲突必须留置或呼叫安保。
悬念：CCTV 一帧出现另一个前台。

#### 第四夜：私人记忆比证件可靠

目标：学会问关系、习惯、问候。
新规则：完美证件不代表安全。
悬念：某位住户被替换，但没人记得他原来的习惯。

#### 第五夜：档案开始污染

目标：学会使用第二来源。
新规则：档案字段可能被改写。
悬念：档案室 CCTV 出现移动的档案卡。

#### 第六夜：伪人学习玩家

目标：不要重复使用同一套验证流程。
新规则：过度完美也是异常。
悬念：蓝星车辆停在楼下，但车牌属于十二年前。

#### 第七夜：证明你是你

目标：证明当前薛夜已经在岗。
新规则：不得登记第二个自己。
悬念：第二个薛夜持完整文件要求换班。

### 执行任务

1. 每夜开头增加 Night Objective Card。
2. 每夜中段加入一次剧情事件。
3. 每夜结尾加入 cliffhanger。
4. 后四个 CCTV 场景按夜晚解锁。
5. 每夜至少有一个视觉新东西，不只是新文字。

---

## 十一、第九阶段：把后 4 个 CCTV 素材接入玩法

### 目标

提高后期视觉升级感。

### 当前问题

项目有 8 个 CCTV 场景，但玩家终端只接入 4 个。后期素材没有变成可玩内容。

### 接入计划

#### 第五夜解锁：污染档案室

用途：

* 查看档案是否被移动。
* 发现某个住户档案卡自己翻页。
* 保存“档案污染”证据。

#### 第六夜解锁：不可能楼梯间

用途：

* 验证不存在七楼。
* 发现楼梯从 6 楼继续向上。
* 识别伪维修工。

#### 第六夜解锁：蓝星维修车辆

用途：

* 查看维修公司车辆。
* 发现车牌和工单年份不符。
* 支撑蓝星维修主线。

#### 第七夜解锁：安保岗亭 / 前台自身监控

用途：

* 验证安保是否失联。
* 验证当前薛夜是否在岗。
* 最终确认第二个薛夜没有影子或没有正确倒影。

### 执行任务

1. CCTV channel list 按夜晚解锁。
2. 每个频道增加 1 到 2 个 hotspot。
3. 点击 hotspot 可冻结并保存证据。
4. 特殊频道出现时播放 unlock 提示。
5. 最终夜强制使用前台监控作为关键证据之一。

---

## 十二、第十阶段：减少文字密度

### 目标

让信息更易读、更像游戏。

### 当前问题

你有大量优秀设定，但游戏内不能一次展示太多。
玩家在紧张判断时，不适合读长段落。

### 改进原则

每个面板只保留一个核心问题。

#### 来客台词

不要超过 2 行。

错误：

“晚上好，我是 302 的李梅，我刚从花店下班，因为今天下雨所以回来晚了，你可以帮我开门吗？”

改成：

“我是 302 的李梅。花店刚关门，让我进去。”

#### 电话结果

不要写长解释。
用短句 + 氛围。

例子：

“302 接通。屋内有人说：‘李梅已经回来了。’”

#### CCTV 结果

不要描述太多，交给画面和 hotspot。

例子：

“电梯显示：7F。”

#### 规则

每日规则最多 4 条。
每条不超过 18 个字。

### 执行任务

1. 全部游戏内文案做一次长度审计。
2. 来客台词控制在 1 到 2 句。
3. 工具结果控制在 1 到 3 行。
4. 结算解释控制在 5 行以内。
5. 长剧情放在夜晚开头或结尾，不放在每个操作结果里。

---

## 十三、第十一阶段：强化工具的“使用成本”

### 目标

提高互动决策感。

### 当前问题

如果玩家可以无成本点完所有工具，那么玩法会变成“把所有按钮点一遍，然后读结果”。

### 改进方案

每晚给工具轻量限制，但不要太苛刻。

#### 普通难度建议

电话：每晚 8 次。
扫描：每晚 6 次。
CCTV 定格截图：每晚 5 次。
提问：每个访客 3 次。
留置：每晚 3 次无惩罚，之后增加压力。
安保：错误呼叫会降低声誉和安保信任。

### 工具选择变成策略

玩家需要决定：

“这个人是证件可疑，还是行为可疑？”

如果证件可疑，用扫描。
如果行为可疑，用提问。
如果位置可疑，用 CCTV。
如果身份重复，用电话。

### 执行任务

1. 在 UI 显示工具资源条。
2. 第一夜资源宽松。
3. 第三夜开始资源有意义。
4. 无尽模式和挑战模式资源更少。
5. 结算显示玩家是否过度依赖某个工具。

---

## 十四、第十二阶段：加入动态提示系统

### 目标

帮助玩家但不破坏推理。

### 提示分三层

#### 轻提示

触发：玩家 15 秒无操作。

例子：

“她说自己住 302。档案夹里应该有记录。”

#### 中提示

触发：玩家 30 秒无操作，或反复切换无进展。

例子：

“试着对比证件姓名和住户档案姓名。”

#### 强提示

触发：第一夜教程或玩家连续错误。

例子：

“点击 Resident ID，再点击 Archive 中的姓名。”

### 提示不能直接告诉答案

不要说：

“她是假的。”

要说：

“目前证据还不足，最好再确认一个来源。”

### 执行任务

1. 新增 hintLevel。
2. 根据 idleTime、wrongActions、currentObjective 触发。
3. 玩家可关闭提示。
4. 设置里可选：完整教程 / 轻提示 / 无提示。
5. 第一夜默认完整教程。

---

## 十五、第十三阶段：强化反馈和音画冲击

### 目标

让每个关键操作都有游戏感。

### 必补反馈

#### 1. 来客说话状态

提问后，来客必须进入 talking 状态。
台词逐字出现。
回答结束后回到 idle / nervous / suspicious。

#### 2. 电话死寂

无人接听不能只是文字。
应该有：

* 听筒暗下
* 低频背景
* log 慢慢出现
* Stability 轻微下降

#### 3. 扫描通过和错误

扫描通过：绿色 check。
扫描错误：红色 error code + 小震动。

#### 4. CCTV 定格

点击异常区域时画面 freeze，出现框线。

#### 5. 正确裁决印章

正确时盖下 Approved / Contained / Cleared 章。
错误时章歪掉或墨迹扩散。

#### 6. 安全和声誉受损

安全下降：楼层背景变暗、远处尖叫。
声誉下降：管理处电话响、投诉提示。
稳定下降：灯光闪烁、UI 漂移。

### 执行任务

1. 补齐 P1 动画反馈。
2. 所有工具结果都绑定视觉状态。
3. 所有数值下降都有专属动画。
4. 裁决结果不只弹窗，要先播放 0.6 到 1 秒小过场。
5. 给重要动画加轻音效占位，即使先用简单 beep 也可以。

## 十六、第十四阶段：改善白天准备阶段

### 目标

让 Prep 不像网页菜单，而像“上班前的桌面”。

### 当前问题

白天准备页结构对，但视觉仍像设置页面。

### 新设计

1. Seven-Night Story：正式合同。

#### Perk 选择

3. Manual Lock Override：门锁手册。
4. Switchboard Kit：电话接线盒。

#### Start Night Shift

改成：

“值班铃”或“门禁总开关”。

点击后：

灯光熄灭 → 夜晚标题 → 电话铃 → 第一位来客敲窗。

### 执行任务

1. 给 Prep 阶段补 10 个关键 sprite。
2. 把模式 card 改成合同纸。
3. 把 perk card 改成桌面物品。
4. Start 按钮改成值班铃。
5. Prep 到 Night 做完整转场。

## 十九、最终推荐的玩家前 10 分钟体验

### 第 0 分钟

玩家看到：

“你是月影公寓午夜门岗。今晚处理 8 位来客。”

页面只突出一个按钮：

“开始第一夜值班。”

### 第 1 分钟

灯光熄灭，Night 1 标题出现。
第一位来客敲窗。
系统提示：

“先查看她的证件。”

### 第 2 分钟

玩家点击证件，纸张滑入。
系统提示：

“打开住户档案，对比姓名。”

玩家发现姓名不符。

### 第 3 分钟

玩家标记证据。
系统提示：

“证据足够。拒绝进入。”

玩家拒绝，播放落闸动画。

### 第 4 分钟

结算告诉玩家：

“正确。她的姓名与档案不符。”

### 第 5 分钟

第二位来客出现。
这次系统只提示：

“检查照片备注。”

玩家学习观察痣、手套、随身物。

### 第 7 分钟

第三位来客出现。
系统教会安保和拒绝的区别。

### 第 10 分钟

玩家已经理解：

1. 怎么查资料。
2. 怎么找矛盾。
3. 怎么标记证据。
4. 怎么裁决。
5. 错误会影响安全、声誉和稳定度。

这时游戏才真正开始自由化。

## 二十、最终结论

《午夜登记簿》现在不缺系统，不缺设定，也不缺素材。
它缺的是：

1. 清晰的新手目标。
2. 被导演过的前 10 分钟体验。
3. 把文字信息转化成可操作证据。
4. 强反馈的工具和裁决动画。
5. 每夜明确的目标、悬念和视觉升级。
6. 避免第三夜提前使用第七夜高潮。

下一阶段不要继续横向加玩法。
应该纵向打磨这条核心体验：

来客敲窗
→ 玩家查看文件
→ 对比档案
→ 使用工具
→ 发现矛盾
→ 标记证据
→ 做出裁决
→ 看到后果
→ 下一位来客带来更深的谜团

只要这条循环变清晰、变有手感、变有反馈，游戏的 engagement 会立刻提升。
