当然可以。你第一版如果已经有了 **访客出现 → 查资料 → 放行/拒绝**，第二版就不要只是“加更多角色”，而是要加：

> **更多互动工具 + 更复杂的判断流程 + 更强的恐怖反馈 + 更有变化的关卡规则。**

下面是一个适合你第二版的设计方案。

# 第二版目标

## V1 核心

```text
看资料 → 判断真假 → 放行/拒绝
```

## V2 核心

```text
看人 → 收文件 → 查档案 → 问问题 → 打电话 → 找证据 → 填处理单 → 承担后果
```

第二版要让玩家感觉自己不是在点选择题，而是真的在做夜班门卫。

---

# V2 新增玩法一：证据板系统

玩家发现可疑点后，可以把证据标记出来。

比如访客说：

```text
我是 302 的林安娜，刚下夜班回来。
```

玩家可以标记：

```text
[证件名字正确]
[房号正确]
[照片不一致]
[今日排班不一致]
[电话无人接听]
```

最后处理时，玩家需要选择原因：

```text
处理原因：
[ ] 证件错误
[ ] 外貌不符
[ ] 今日名单不符
[ ] 电话确认失败
[ ] 行为异常
```

这样比直接按“拒绝”更有互动感。

好处是：
玩家不是乱猜，而是在建立“证据链”。

---

# V2 新增玩法二：提问系统

给玩家每个访客最多 **3 次提问机会**。

问题可以分成几类：

## 身份问题

```text
你住几号房？
你的全名是什么？
你的职业是什么？
你什么时候入住的？
```

## 关系问题

```text
你隔壁住户是谁？
你认识管理员王叔吗？
你上次维修是什么时候？
```

## 情境问题

```text
你为什么这么晚回来？
你为什么没有带门禁卡？
你今天见过谁？
```

## 陷阱问题

这类问题专门用来抓伪人。

例如档案写：

```text
林安娜讨厌别人叫她“小林”。
```

玩家可以问：

```text
小林，你今天几点下班？
```

真实林安娜会生气：

```text
“我说过不要这样叫我。”
```

伪人可能正常回答：

```text
“我十点下班。”
```

这就暴露了。

---

# V2 新增玩法三：电话系统升级

第一版可能只是“打电话确认”。第二版可以做成多线路电话。

玩家可以拨：

```text
房间电话
管理处
安保室
医院/公司/维修单位
紧急联系人
```

不同电话会给不同线索。

## 例子

访客声称是医生。

玩家可以打：

```text
302 房间电话
医院值班室
管理处登记处
```

可能结果：

```text
302：无人接听
医院：她今晚没有值班
管理处：302 住户今天 18:30 已经进楼
```

这时候玩家就知道门口的人很可疑。

---

# V2 新增玩法四：访客情绪反应

访客不应该只是站着不动。玩家操作不同，访客要有反应。

## 玩家查看证件太久

```text
“你看够了吗？”
“我真的很累。”
“外面很冷，快点让我进去。”
```

## 玩家打电话

```text
“没必要打电话吧？”
“他们现在肯定睡了。”
“你不相信我？”
```

## 玩家问到关键问题

```text
真实住户：正常回答
普通伪人：沉默几秒
高级伪人：转移话题
危险伪人：开始威胁玩家
```

## 玩家准备报警

```text
“等等。”
“你会后悔的。”
“我知道你住在哪里。”
```

这会让游戏更有压迫感。

---

# V2 新增玩法五：访客类型扩展

第一版可以只有住户。第二版可以加入不同身份。

## 1. 住户

需要检查：

```text
姓名
房号
外貌
习惯
出入记录
```

## 2. 访客

需要检查：

```text
预约人
预约时间
访问房间
访客证件
是否在今日名单
```

## 3. 维修工

需要检查：

```text
公司名
维修单
维修区域
工具箱
管理处通知
```

## 4. 外卖员

需要检查：

```text
订单号
收件人
房号
是否允许上楼
时间是否正确
```

## 5. 紧急人员

比如医生、警察、消防员。

这类最容易制造压力，因为玩家会犹豫：

```text
如果是真的，不放进去可能害死人。
如果是假的，放进去可能全楼出事。
```

# V2 新增玩法六：每日规则

每天开始前给玩家一张通知。

例如：

```text
第 2 夜通知：

1. 今晚 402 住户不在家。
2. 所有维修人员必须出示蓝色维修单。
3. 外卖员不得进入电梯。
4. 访客必须在 22:00 前登记。
5. 如果有人声称来自 6 楼，必须电话确认。
```

这样每一天的判断逻辑都会变。

玩家不能只记住居民资料，还要看当天规则。

---

# V2 新增玩法七：出入记录系统

增加一本“今日出入记录”。

玩家可以查：

```text
谁已经进楼
谁还没回来
谁今天出门了
谁今晚请假离开
```

这个系统很好玩，因为可以制造这种情况：

```text
门口的人说：我是 501 的陈医生。
出入记录显示：陈医生 19:20 已经进楼，未离开。
```

那门口这个人就很可能是假的。

但也可以有反转：

```text
陈医生可能从后门出去了，但系统没记录。
```

所以玩家需要结合电话和提问判断。

---

# V2 新增玩法八：伪人等级

第二版可以把伪人分等级。

## 低级伪人

明显错误：

```text
名字错
房号错
照片不像
证件过期
```

## 中级伪人

细节错误：

```text
痣的位置反了
眼睛颜色不对
职业说错
签名不一致
预约时间差 30 分钟
```

## 高级伪人

证件和外貌都对，但行为不对：

```text
不知道邻居是谁
不记得自己的宠物
说出错误的生活习惯
回答太机械
```

## 学习型伪人

它会根据玩家习惯变化。

比如玩家总是先查证件，它的证件就会完美，但电话或行为会出错。

# V2 新增玩法九：错误后果更明显

第一版可能只是加分扣分。第二版可以让错误影响后续剧情。

## 放进伪人

后果可能是：

```text
某个住户失踪
第二天档案被污染
电话系统出现杂音
出现新的伪人类型
玩家精神值下降
```

## 错拒真人

后果可能是：

```text
住户投诉
管理处警告
信誉值下降
某些住户之后不愿意配合
真实访客开始更急躁
```

## 乱报警

后果可能是：

```text
安保信任度下降
错误报警太多后，安保响应变慢
关键时刻报警可能延迟
```

这样玩家会更认真。

# 一句话总结 V2

第二版应该从“判断真假游戏”升级成：

> **交互式夜班门卫模拟器：玩家通过文件、电话、提问、出入记录和每日规则建立证据链，再决定是否放人进楼。**

我建议你 V2 优先做 **提问系统 + 电话系统 + 出入记录系统**。这三个加上去，互动性会立刻比第一版强很多。


动画部分建议你不要一上来做很复杂。这个类型的游戏最重要的不是角色走路、战斗动画，而是：

> **让玩家每一次操作都有反馈，让访客看起来“活着”，让错误决定有压迫感。**

你可以把动画分成 4 层来做。

---

# V2 动画设计方向

## 1. 访客动画

访客可以不用完整 Live2D 或骨骼动画，最简单有效的是：

```text
静态立绘 + 轻微呼吸动画 + 表情切换 + 偶尔小动作
```

### 访客基础状态

每个访客准备 4 到 6 张图就够：

```text
normal.png       正常表情
blink.png        眨眼
nervous.png      紧张
angry.png        生气
smile.png        诡异微笑
reveal.png       暴露伪人
```

### 动画效果

| 场景      | 动画                   |
| ------- | -------------------- |
| 访客出现    | 从黑暗中淡入 / 从下方滑入窗口     |
| 等待中     | 轻微上下浮动，模拟呼吸          |
| 玩家检查太久  | 眼睛眨动变快，表情变烦躁         |
| 玩家打电话   | 访客突然停止呼吸动画           |
| 玩家问关键问题 | 表情切到 nervous / smile |
| 玩家报警    | 访客敲窗、画面震动            |
| 伪人暴露    | 脸部闪一下，切换怪异图          |

---

# 2. 文件/证件动画

这个很重要，因为玩家大部分时间都在看文件。

## 文件递交动画

访客出现后，把文件“推”到桌面上：

```text
文件从窗口下方滑入桌面
落下时轻微旋转
发出纸张声
```

## 文件点击放大

玩家点击证件：

```text
证件放大到屏幕中央
背景稍微变暗
鼠标移动时文件轻微倾斜
```

## 文件拖动

可以让玩家拖证件到档案旁边对比。

```text
拖动时文件有阴影
松手时轻微弹回
错误文件可以有轻微红光提示，但不要太明显
```

不要自动告诉玩家哪里错，只给操作反馈。

---

# 3. UI 按钮动画

按钮反馈一定要做，因为它会让游戏更有手感。

## 开门按钮

```text
鼠标悬停：绿色灯微亮
点击：按钮下压
成功放行：门锁灯从红变绿
错误放行：灯闪烁一下，然后恢复
```

## 拒绝按钮

```text
点击后窗口玻璃变暗
访客后退消失
纸张被收走
```

## 报警按钮

这是最有氛围的按钮。

```text
点击红色按钮
屏幕轻微震动
红灯闪烁
警报声响
窗口铁板落下
访客被黑影带走 / 画面切黑
```

不需要真的做复杂角色动画，只要：

```text
红光闪烁 + 屏幕震动 + 音效 + 访客图快速消失
```

效果就会很好。

---

# 4. 恐怖反馈动画

这是最能提升质感的部分。

## 伪人接近暴露时

当玩家问到关键问题，可以触发小异常：

```text
访客眼睛轻微变大
嘴角上扬 0.5 秒
画面闪一下
背景音突然变低
文字框出现乱码一瞬间
```

## 玩家判断错误时

如果放进伪人：

```text
门打开
访客进入
灯闪烁
远处传来尖叫
第二天某个住户档案变灰
```

如果误拒真人：

```text
访客生气离开
管理处电话响起
信誉值下降
```

---

# 最推荐的动画优先级

如果你是一个人开发，我建议按这个顺序做。

## 第一优先级：必须做

```text
1. 访客出现/离开动画
2. 文件滑入桌面动画
3. 文件点击放大动画
4. 按钮按下反馈
5. 开门/拒绝/报警动画
6. 电话拨号等待动画
```

这些会直接影响游戏手感。

## 第二优先级：提升氛围

```text
1. 访客眨眼
2. 访客呼吸
3. 访客表情切换
4. 红灯闪烁
5. 屏幕震动
6. 文字框打字机效果
```

这些会让游戏更像完整作品。

## 第三优先级：后期再做

```text
1. 复杂脸部变形
2. 骨骼动画
3. 完整身体动作
4. Live2D
5. 复杂过场动画
```

这些很耗时间，不适合第二版一开始做。

---

# 一个完整访客的动画流程

比如一个访客来到窗口。

```text
1. 背景门铃响
2. 访客从暗处淡入
3. 台词框打字机显示：“晚上好，我是 302 的林安娜。”
4. 文件从窗口下方滑到桌面
5. 玩家点击身份证，证件放大
6. 玩家打开档案，档案页翻开
7. 玩家拨电话，电话按钮闪烁
8. 访客表情从 normal 切到 nervous
9. 玩家按报警
10. 红灯闪烁，窗口震动，铁板落下
11. 访客图像消失
12. 进入下一个访客
```

这个流程不需要复杂动画，但会非常有互动感。

# 访客表情切换逻辑

你可以用状态控制图片。

```js
const visitorStates = {
  idle: "normal.png",
  waiting: "nervous.png",
  suspicious: "smile.png",
  angry: "angry.png",
  reveal: "reveal.png"
};
```

触发条件：

```js
if (playerChecksTooLong) {
  setVisitorState("waiting");
}

if (playerCallsPhone) {
  setVisitorState("suspicious");
}

if (playerPressesAlarm) {
  setVisitorState("reveal");
}
```

这样不用做复杂动画，只要换图 + CSS 动画就够。

# 最重要的建议

不要把动画重点放在“角色动得多复杂”。
这个类型最需要的是：

> **操作反馈动画 + 氛围动画 + 表情状态变化。**

也就是说，第二版动画目标应该是：

```text
让玩家点每一个东西都有反馈；
让访客在等待时有压迫感；
让报警、开门、错误决定有情绪冲击。
```

这样就很接近 That’s Not My Neighbor 的感觉，而且开发量不会太大。

并且把这些新的元素都加入到design system。 所以可以重复使用
